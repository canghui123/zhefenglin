# 库存决策沙盘批量导入

## 本轮能力

- 单台沙盘支持车300估值留空：系统优先用 VIN 调用车300估值，缺 VIN 时回退到系统估值模型。
- 批量导入支持 `.xlsx` / `.xls` / `.csv`：自动识别车辆、VIN、入库日期、逾期金额、逾期阶段、车辆状态、区域、估值等字段。
- 批量导入后进入可编辑预览表：缺字段行会标记，用户可逐台补填，也可取消勾选。
- 竞拍参数从“竞拍佣金比例”调整为“竞拍折扣比例”：留空时系统建议折扣，并在前端询问用户是否接受。
- 分期重组/和解的再违约率可留空：此时必须输入客户过往催收或逾期记录，系统根据记录给出建议值并询问用户是否接受。
- 库存沙盘表单移除“寻车线索评分”，寻车线索仍保留在外部数据网关中，后续用于派单优先级。
- 批量运行会生成可复查批次号，用户可进入 `/inventory-sandbox/batches/{batchId}` 查看每台车完整五路径结果、失败原因，并可针对单台生成报告。

## API

- `POST /api/sandbox/suggestions`
  - 输入车辆、逾期、估值、催收记录，返回竞拍折扣和再违约率建议。
- `POST /api/sandbox/import-preview`
  - 上传客户表格，返回可编辑的沙盘输入行、缺字段和自动估值状态。
- `POST /api/sandbox/batch-simulate`
  - 对已勾选且补齐字段的车辆批量运行五路径沙盘，返回 `batch_id`，并保存每台车的沙盘结果。
- `GET /api/sandbox/batches`
  - 查询当前租户最近批量模拟批次。
- `GET /api/sandbox/batches/{batch_id}`
  - 查询某个批次的逐车明细、推荐路径、失败原因和完整五路径结果。

## 字段识别示例

- 车辆：车辆描述、车型、品牌型号、车辆型号
- VIN：VIN、车架号、车辆识别代码
- 日期：入库日期、评估日期、首次上牌、上牌日期
- 逾期：逾期阶段、逾期天数、逾期金额、风险敞口
- 估值：当前车300估值、车300估值、车辆估值、市场价
- 状态：是否收回、收车状态、是否入库、入库状态
- 重组：月还款额、重组期数、再违约率、过往催收记录

## 业务边界

批量导入只对用户勾选的车辆运行模拟；缺少车辆描述、入库/评估日期、逾期金额或估值且无法自动估值的车辆不会进入模拟。每次批量运行会同时写入 `sandbox_simulation_batches` 和 `sandbox_simulation_batch_items`，成功行仍按租户隔离进入原有 `sandbox_results` 台账。
