# 客户数据接入中心

## 目标

客户原有系统通常分散在核心贷后系统、GPS/寻车平台、法务系统和人工 Excel 台账中。数据接入中心先把这些导出文件统一进入可审计的 staging 台账，再由业务确认后进入资产池、动作中心或模型复盘。

## 支持格式

- `.xlsx`
- `.xls`
- `.csv`，优先按 `utf-8-sig` 解析，失败后回退 `gbk` / `gb18030`

## 首批识别字段

- 资产编号：资产编号、资产ID、案件编号、账户编号、客户编号
- 合同号：合同编号、合同号、借据号、贷款合同号
- 客户：客户姓名、债务人、借款人、姓名
- 车辆：车型、品牌型号、车辆、车辆描述、车辆型号
- VIN/车牌：VIN、车架号、车辆识别代码、车牌号
- 区域：省份、城市、资产所在地、地区、车辆所在地
- 逾期：逾期阶段、逾期分段、M值、逾期天数、逾期金额、风险敞口
- 价值：剩余本金、贷款余额、车辆估值、市场价、评估价
- 状态：车辆状态、收车状态、是否收回、入库状态
- 定位：GPS时间、定位时间、最近定位

## API

- `POST /api/data-import/upload`
  - form-data: `file`, `source_system?`, `import_type?`
  - 返回导入批次、前 20 行预览、识别字段和未映射字段。
- `GET /api/data-import/batches`
  - 返回当前租户最近导入批次。
- `GET /api/data-import/batches/{batch_id}/rows?status=valid|error`
  - 返回批次明细，可按可用/错误行筛选。
- `POST /api/portfolio/source/clear`
  - 将当前租户活跃的 `asset_ledger` 批次从组合分析数据源中归档，组合页进入空状态。
  - 只更新批次状态为 `archived`，不删除 `data_import_batches` 或 `data_import_rows` 历史记录。

## 数据边界

- 上传文件走统一存储抽象，生产可落到 MinIO/S3。
- 每行同时保存 `raw_json` 和 `normalized_json`，避免字段映射错误时丢失原始证据。
- 组合总览、分层分析、路径模拟、现金回流、管理决策和动作中心会优先读取当前租户最新一批 `asset_ledger` 可用行；没有资产台账导入时，才回退演示组合。
- 如果租户曾经导入过资产台账但当前没有活跃的 `parsed` 批次，组合页返回空组合，避免继续展示旧数据或演示数据。
- 可用行会按逾期阶段和车辆状态聚合为组合分层，并保留行级资产信息供拖车/拍卖工单候选车辆展开。
- 原始导入不会直接覆盖资产池主表，后续仍可增加“确认入池”动作，把接入台账转为长期资产池。
- 所有批次和明细都按 `tenant_id` 隔离。
