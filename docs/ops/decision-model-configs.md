# 处置决策模型配置说明

本批升级将库存沙盘、资产包定价和现金回流的关键商业口径沉淀到可配置模型层，避免页面硬编码规则。

## 配置表

- `brand_retention_profiles`：品牌/车型保值模型，控制保值率系数、月折旧曲线、新能源技术迭代折价。
- `region_disposal_coefficients`：区域处置系数，控制二手车流通速度、法务执行效率和拖车成本修正。

## 默认数据

默认配置由 `backend/scripts/seed_decision_model_defaults.py` 写入，脚本幂等，可重复执行。

```bash
cd backend
python3 -m scripts.seed_decision_model_defaults
```

在 seed 未运行时，后端服务层仍有内置兜底配置，确保沙盘和定价链路可运行。

## 业务口径

- 库存沙盘按动态成功概率计算路径结果，输入包括车龄、逾期金额/估值比、品牌保值率、车辆占有状态、区域系数。
- 路径 D“实现担保物权特别程序”必须满足车辆已收回、已入库、逾期 M3 以上；不满足时成功率为 `0` 且不进入推荐候选。
- 推荐排序使用 `future_marginal_net_benefit`，已发生催收/法务成本作为 `sunk_cost_excluded` 展示，不参与路径优劣比较。
- 资产包定价按区域修正拖车费、处置周期和预期收入折扣；新能源车使用更陡折旧并附带风险标签。
- 现金回流保留 `net_cash_flow` 兼容旧接口，同时新增悲观/中性/乐观三情景字段。

## 后续接入方向

- 后台管理页可直接维护上述两张配置表。
- 后续如引入历史成交数据，可先替换 `services/decision_model.py` 中的规则模型，不需要改页面消费口径。
